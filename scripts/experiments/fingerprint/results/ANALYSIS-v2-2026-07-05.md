# Analysis — fingerprint v2 recall run (2026-07-05, same day as v1)

**Verdict: pre-registered criterion NOT MET a second time (recall@25 =
5/12 = 0.42, identical to v1). Two design iterations in one day both
plateaued at 0.42. Per the project's measurement discipline, the honest
conclusion is that pure fingerprint-similarity ranking is not a
top-25-precision matcher — and the data now says clearly what it IS.**

Fingerprinting model: Claude (Fable 5) subagents, v2 schema
(multi-view with `deterministic_skeleton`), 54 papers (12 primary planted
pairs + 3 vocabulary-blind held-out pairs + 23 distractors + 1 retired v1
paper kept as a hard distractor), 1,285 cross-archive candidate pairs.

Note on comparability: the v2 candidate pool is larger than v1's (1,285
vs 966), so a fixed K=25 is a *stricter* bar in v2 (top 1.9% vs 2.6%).
Normalizing to v1's fraction does not change the verdict (still 5/12).

## What the two failures together actually say

1. **The skeleton view is a double-edged sword.** It rescued exactly the
   pairs it was designed for (lotka_volterra 122→12, oscillator, logistic,
   replicator all improved) but *diluted* pairs whose shared signature IS
   the dressing (kuramoto's phase coupling, fokker-planck's noise
   structure), and max-over-views also lifts distractor scores (skeletons
   collapse diversity), squeezing true pairs out of the top 10.
   recall@10 fell 5/12 → 3/12 even as the tail improved.

2. **The persistent misses are largely a gold-standard realization
   problem, not a model-comprehension problem.** Research papers are
   about *twists* on canonical models. The two-temperature Ising paper
   and the "Universal Hopfield Networks" paper genuinely do NOT have the
   same core model — the canonical models *behind* them do. A pairwise
   paper-to-paper matcher is being asked to see through two layers of
   variation at once. Where both papers sit close to a canonical skeleton
   (transport↔matching, sine-Gordon↔Josephson), the fingerprints match
   *perfectly* (both 1.000).

3. **As a top-25 matcher the substrate fails; as a candidate BLOCKER it
   looks strong.** Blocking-recall curve (primary pairs, 1,285 candidates):
   recall@100 = 7/12, **recall@300 = 11/12** (top 23%). Everything except
   ising_hopfield lands in the top quarter of the ranking. A two-stage
   design — fingerprint blocking to ~top-20%, then LLM pairwise
   verification of the block — would have caught 11/12 known isomorphisms
   at this corpus size.

## The two coherent paths forward (decision for the admin)

**Path A — two-stage matcher.** Keep fingerprints as the blocker; add an
LLM pairwise judge over the blocked candidates ("same core structure?
what maps to what?"), which drafts the Tier-2 correspondence write-up as
a side effect. Cost scales with block size (~20% of cross-domain pairs at
this density — needs re-measurement at realistic corpus size before
believing). Requires a NEW pre-registered criterion (e.g. end-to-end
recall ≥ 0.75 on this same gold standard, judged precision ≥ some floor
on a distractor sample) — the original recall@25 criterion has now
failed twice and should not be quietly replaced by a friendlier metric
without the admin signing off on the new one.

**Path B — canonical-template indexing (the "atlas" architecture).**
Stop matching papers to papers. Build a library of ~50–100 named
canonical structures (logistic, sine-Gordon, Kuramoto, Fokker–Planck,
replicator, ...); fingerprint each paper against the library ("which
canonical skeleton(s) does this paper's model instantiate, and with what
twist?"); cross-domain analogies then fall out of the index (all papers
under one skeleton, grouped by field). This matches finding 2 directly,
turns the product into the atlas the project pivot memo described, and
makes every match explainable ("both instantiate X") rather than
similarity-scored. Risk: it finds instances of *known* structures by
construction; genuinely novel structural coincidences need the pairwise
path. (The two paths compose: B for the atlas, A for the novel tail.)

## Bottom line

Do not run a v3 of the same shape. Two same-day iterations landed on the
same number; the marginal fix is not another scoring tweak. The substrate
has a real, measured capability — near-perfect matching for
canonical-adjacent papers, 11/12 blocking recall at top-23% — and a real,
measured limitation — it cannot deliver top-25 precision alone. The next
step is an admin decision between Path A, Path B, both, or parking with
the re-run-this-experiment-with-a-newer-model tripwire documented in
HANDOFF.
